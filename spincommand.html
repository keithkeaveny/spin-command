⚠ Why Spotify rejects the connection
Spotify only allows https:// redirect URIs — opening a local file won't work on iPad. The fix is to host this file online for free using GitHub Pages. It takes about 5 minutes and is free forever.
1
Create a free GitHub account at github.com — or sign in if you already have one.
2
Tap the + icon (top right) → New repository.
Name it spin-command · set to Public · tick "Add a README file" · tap Create repository.
3
Inside your new repository, tap Add file → Upload files.
Upload spincommand.html from your Files app, then tap Commit changes.
4
Tap Settings (in your repo) → scroll to Pages in the left menu.
Under Source, choose Deploy from a branch · branch: main · folder: / (root) · tap Save.
Wait about 60 seconds. Refresh the page and your live URL will appear — it looks like:
https://yourusername.github.io/spin-command/spincommand.html
5
Go to developer.spotify.com/dashboard → your app → Settings → Redirect URIs.
Add your full GitHub Pages URL exactly (copy it from Step 4), tap Add then Save.
6
Open your GitHub Pages URL in Safari or Chrome on your iPad.
Tap CONNECT SPOTIFY — it will work. ✓
Bookmark that URL so you can open it quickly before each class. You need Spotify Premium for playback control.
This must match exactly what you add in your Spotify app's Redirect URIs. Paste it here so the app uses it when authorising.
Your Client ID is already embedded. Only change this if you create a new Spotify app.
ℹ How It Works
Once connected, Spin Command can:
• Play / pause / skip tracks directly from the console
• Auto-switch playlists when the zone changes during class
• Show album art, track name, artist and BPM in real time
• Control volume from the music panel

Playback happens through your existing Spotify session — phone, speaker, or this browser tab. Requires Spotify Premium.
